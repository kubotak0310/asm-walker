フラグレジスタ

演算結果の「性質」を1ビットで記録する特殊なレジスタ群を フラグレジスタ と呼びます。 条件分岐命令はこのフラグを読み取ってジャンプするかどうかを決定します。 ARM と x86 にはフラグレジスタがありますが、RISC-V には存在しません。

目次

  1. ARM フラグ（CPSR: N / Z / C / V）
  2. x86-64 フラグ（RFLAGS: ZF / CF / SF / OF）
  3. RISC-V のフラグなし設計
  4. フラグと条件分岐の関係

1. ARM フラグ（CPSR）

ARM Cortex-M / CPSR

ARM は CPSR（Current Program Status Register） の上位4ビットにフラグを持ちます。 ADDS・SUBS・CMP・TST などの命令で更新され、BEQ・BLT などの条件分岐命令で参照されます。

フラグ名前セットされる条件典型的な使用場面
N Negative 負フラグ 演算結果の最上位ビット（ビット31）が 1（符号付き演算で結果が負） BMI（結果が負ならジャンプ）
Z Zero ゼロフラグ 演算結果がちょうど 0 BEQ（等しい）・BNE（等しくない）
C Carry キャリーフラグ 加算で 32bit からあふれる繰り上がり発生、または減算で借りが発生しなかったとき BCS（繰り上がりあり）・符号なし比較（BHI/BLS）
V Overflow オーバーフローフラグ 符号付き演算で結果が -2³¹〜2³¹-1 の範囲を超えた（正+正=負、または負+負=正になった） BVS（オーバーフロー）・符号付き比較（BGT/BLE）
ARM の C フラグは x86 と向きが逆：
ARM の減算（SUB/CMP）で C フラグは「借りが発生しなかった」ときに 1 になります。 x86 の CF は「借りが発生した」ときに 1 になるため、向きが逆です。 AsmWalker の特殊レジスタパネルには C フラグの値がリアルタイムで表示されます。

命令に付けるフラグ更新サフィックス

ARM は命令名に S を付けると演算結果でフラグを更新します。付けない場合はフラグは変化しません。

MOV  r0, #0     ; r0 = 0。フラグは変わらない
MOVS r0, #0     ; r0 = 0。結果が 0 なので Z=1, N=0 に更新される

CMP  r0, r1     ; r0 - r1 を計算してフラグだけを更新（S なしでも常に更新）
BEQ  label      ; Z=1 ならジャンプ（= r0 == r1）
AsmWalker での確認：
ARM サンプル（条件分岐）をコンパイルしてステップを進めると、CMP 命令のステップで特殊レジスタパネルのフラグが更新されるのが確認できます。 その後の BEQ/BNE のステップでどのフラグが参照されてジャンプが起きるかも追跡できます。

2. x86-64 フラグ（RFLAGS）

x86-64 / RFLAGS

x86-64 は RFLAGS レジスタ（64bit）に多数のフラグを持ちます。アセンブラコードで頻繁に使われる主要フラグは4〜5個です。 CMP・TEST・ADD・SUB などで更新され、JE・JL などの条件ジャンプで参照されます。

フラグ名前セットされる条件典型的な使用場面
ZF Zero Flag 演算結果がちょうど 0 JE（等しい）・JNE（等しくない）
SF Sign Flag 演算結果の最上位ビットが 1（負の数） JS（負）・符号付き比較との組み合わせ
CF Carry Flag 符号なし演算で桁あふれ（加算で繰り上がり・減算で借りが発生） JB（符号なし未満）・JAE（符号なし以上）
OF Overflow Flag 符号付き演算でオーバーフロー（正+正=負、または負+負=正になった） JO（オーバーフロー）・符号付き比較（JL/JGE）
PF Parity Flag 結果の下位8ビット中の 1 の個数が偶数 浮動小数点比較・一部の暗号処理（日常のコードではほぼ不使用）

CMP と TEST の違い

CMP  rax, rbx    ; rax - rbx を計算し、フラグのみ更新（rax は変化しない）
TEST rax, rax    ; rax AND rax を計算し、フラグのみ更新（ゼロチェックによく使われる）

; 条件ジャンプの例
cmp  eax, 0      ; eax - 0 → eax が 0 なら ZF=1
je   .zero       ; ZF=1 なら .zero へジャンプ（= eax == 0）

test eax, eax    ; eax & eax → eax が 0 なら ZF=1（cmp eax, 0 と等価だが1バイト短い）
jne  .nonzero    ; ZF=0 なら .nonzero へジャンプ（= eax != 0）
AsmWalker での確認：
x86 サンプル（条件分岐）をコンパイルしてステップを進めると、CMP/TEST 命令のステップで特殊レジスタパネルのフラグ（ZF/SF/CF/OF）が更新されます。 その後の JE/JL 等のステップでジャンプが実際に起きるかどうかが確認できます。

3. RISC-V のフラグなし設計

RISC-V RV32 / フラグレジスタなし
RISC-V にフラグレジスタは存在しません。

ARM や x86 は「比較命令でフラグをセット → 条件ジャンプ命令でフラグを読む」という2ステップを踏みます。 RISC-V は 1命令で比較とジャンプを同時に行います。
アーキ「a == b ならジャンプ」の実装
ARM CMP r0, r1（r0 - r1 でフラグ更新）
BEQ label（ZF=1 ならジャンプ）
x86-64 CMP eax, ebx（eax - ebx でフラグ更新）
JE label（ZF=1 ならジャンプ）
RISC-V beq a0, a1, label（a0 == a1 なら label へ — 1命令で完結）

RISC-V の分岐命令（beq, bne, blt, bge, bltu, bgeu）はすべて「2つのレジスタを比較してジャンプ」という形式です。フラグを経由しないため、AsmWalker の特殊レジスタパネルにフラグ欄は表示されません。

命令条件備考
beq rs1, rs2, labelrs1 == rs2Branch if Equal
bne rs1, rs2, labelrs1 != rs2Branch if Not Equal
blt rs1, rs2, labelrs1 < rs2（符号付き）Branch if Less Than
bge rs1, rs2, labelrs1 ≥ rs2（符号付き）Branch if Greater or Equal
bltu rs1, rs2, labelrs1 < rs2（符号なし）Branch if Less Than Unsigned
bgeu rs1, rs2, labelrs1 ≥ rs2（符号なし）Branch if Greater or Equal Unsigned
「ゼロと比較する」パターン：
RISC-V の zero レジスタ（常に 0）との比較で、ARM/x86 の「0 との CMP」と同等のことができます。
beq a0, zero, label → a0 == 0 ならジャンプ（ARM の CBZ・x86 の test eax,eax + je 相当）
blt a0, zero, label → a0 が負ならジャンプ（ARM の BMI 相当）

4. フラグと条件分岐の関係

ARM・x86 の条件分岐命令は、直前の演算/比較命令が残したフラグを読んで「ジャンプするかどうか」を決めます。

ARMx86-64条件（C言語的表現）参照フラグ
BEQJEa == bZ=1
BNEJNEa != bZ=0
BLTJLa < b（符号付き）N != V
BLEJLEa ≤ b（符号付き）Z=1 または N != V
BGTJGa > b（符号付き）Z=0 かつ N == V
BGEJGEa ≥ b（符号付き）N == V
BLO / BCCJBa < b（符号なし）C=0（ARM） / CF=1（x86）
BHS / BCSJAEa ≥ b（符号なし）C=1（ARM） / CF=0（x86）
BMIJS結果が負N=1 / SF=1
BPLJNS結果が正または 0N=0 / SF=0
符号付き vs 符号なし比較の使い分け：
同じ「小さい」でも符号付き（BLT/JL）と符号なし（BLO/JB）では参照するフラグが異なります。 コンパイラは C の int（符号付き）には BLT/JL を、unsigned int（符号なし）には BLO/JB を使います。 アセンブラを読む際はどちらの比較が使われているか確認する習慣をつけると理解が深まります。
