## for ループ、アセンブラで書くと何行になるか知っていますか？

```c
for (int i = 0; i < 5; i++) {
    sum += i;
}
```

この4行のCコードをコンパイルすると、アセンブラでは何行になるか。

正解は**約10行**。しかも、`for` という命令はアセンブラには存在しない。

では何があるのか。**比較命令とジャンプ命令だけ**だ。

---

## for ループはアセンブラで「後ろ向きジャンプ」になる

コンパイルされたアセンブラを見てみよう。

```
    mov  r3, #0         ; i = 0
    str  r3, [r7, #0]   ; i をスタックに保存
loop:
    ldr  r3, [r7, #0]   ; i を読み込む
    cmp  r3, #4         ; i と 4 を比較
    bgt  end            ; i > 4 なら終了へジャンプ
    ldr  r2, [r7, #4]   ; sum を読み込む
    ldr  r3, [r7, #0]   ; i を読み込む
    add  r3, r2, r3     ; sum += i
    str  r3, [r7, #4]   ; sum を書き戻す
    ldr  r3, [r7, #0]   ; i を読み込む
    add  r3, r3, #1     ; i++
    str  r3, [r7, #0]   ; i を書き戻す
    b    loop           ; loop ラベルに戻る（後ろ向きジャンプ）
end:
```

最後の `b loop` ——これがループの正体だ。

**ループとは、条件が満たされる間だけ繰り返す「後ろ向きジャンプ」だった。**

---

## cmp と bgt がセットで「条件分岐」を作る

```
cmp  r3, #4    ; r3 と 4 を比較（引き算をしてフラグをセット）
bgt  end       ; フラグが「大きい（Greater Than）」なら end へジャンプ
```

`cmp` は Compare の略。実際には **r3 - 4 を計算**して、結果をフラグレジスタに記録する。値そのものは書き換えない。

`bgt` は Branch if Greater Than の略。フラグレジスタの状態を見て、「大きければジャンプ」する。

この2命令のセットが、C の `if (i < 5)` に相当する。

| C の書き方 | ARM アセンブラ |
|---|---|
| `if (a > b)` | `cmp a, b` + `bgt` |
| `if (a == b)` | `cmp a, b` + `beq` |
| `if (a != b)` | `cmp a, b` + `bne` |
| `if (a <= b)` | `cmp a, b` + `ble` |

---

## while ループも同じ構造

```c
while (i < 5) {
    sum += i;
    i++;
}
```

アセンブラ上では `for` も `while` も区別がない。どちらも「比較 + 条件ジャンプ + 後ろ向きジャンプ」に変換される。

**C の制御構文（for / while / if）はすべて、比較命令とジャンプ命令の組み合わせに過ぎない。**

## コンパイラの最適化で for ループが消えることもある

`-O2` 最適化をかけると、ループ変数がすべて定数の場合、コンパイラは**実行前に計算を済ませて結果だけをコードに埋め込む**。

```c
for (int i = 0; i < 5; i++) {
    sum += i;   // 0+1+2+3+4 = 10 とコンパイル時に計算
}
// -O2 では → add sum, #10 の1命令になる
```

**定数畳み込み（Constant Folding）** と呼ばれる最適化だ。ループもジャンプも消えて、定数の加算1命令だけが残る。

「ループ展開（Loop Unrolling）」など他の最適化も存在するが、どの最適化が適用されるかはコンパイラのバージョンやコードのパターンによって異なる。

AsmWalker で `-O2` に切り替えると、この変化を確認できる。

## まとめ：for ループの正体

```
for (int i = 0; i < 5; i++) { ... }
```

| C の部分 | アセンブラでの実装 |
|---|---|
| `int i = 0` | `mov r3, #0` でレジスタを初期化 |
| `i < 5`（条件チェック） | `cmp r3, #4` + `bgt end` |
| ループ本体 | そのまま実行 |
| `i++` | `add r3, r3, #1` |
| ループに戻る | `b loop`（後ろ向きジャンプ） |

C言語の「制御構文」は人間のための抽象化だ。CPU は条件とジャンプしか知らない。

それを知ったとき、**「なぜ無限ループはCPUを食い続けるのか」「なぜ条件分岐はコストがかかるのか」** が、感覚として理解できるようになる。
