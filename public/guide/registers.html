レジスタ一覧

CPU が直接操作できる超高速な記憶場所を レジスタ と呼びます。すべての計算・メモリアクセス・ジャンプはレジスタを経由して行われます。このページでは ARM・x86-64・RISC-V RV32 のレジスタをABI用途とともに一覧します。

目次

  1. ARM Cortex-M レジスタ（r0〜r15）
  2. x86-64 レジスタ（rax〜r15）
  3. RISC-V RV32 レジスタ（x0〜x31）
  4. Caller-saved と Callee-saved
  5. 引数・戻り値レジスタのまとめ

Thumb モードと FP の位置：
Cortex-M は Thumb 命令セットを使います。Thumb-2 の 2レジスタ命令（push/pop）では r8 以上が使いにくいため、 GCC は -mthumb 時にフレームポインタとして r7 を、ARM モードでは r11 を使います。 AsmWalker のサンプルは Thumb モードでコンパイルするため r7 が FP として現れます。
AsmWalker での確認：
関数呼び出しサンプルをコンパイルし bl 命令のステップまで進めると、 r0〜r3 に引数値がセットされた状態でバッジが表示されます。 bx lr / pop {pc} ステップでは r0 に「戻り値」バッジが表示されます。

32bit 名で書くと上位32bitが自動ゼロになる：
int（32bit）の引数を渡すとき、GCC は rdi ではなく edi を使います。 x86-64 の仕様で、edi に書き込むと rdi の上位32bitが自動的に 0 になるため、 動作は rdi への書き込みと等価です。
AsmWalker での確認：
x86 サンプルをコンパイルし call 命令のステップまで進めると、 rdi/rsi/rdx などに引数値がセットされた状態でバッジが表示されます。 ret ステップでは rax（または eax）に「戻り値」バッジが表示されます。

zero レジスタ（x0）の活用：
常に 0 を返す専用レジスタを持つことで、多くの疑似命令をシンプルに実現できます。
例：mv a0, a1 → 実際は addi a0, a1, 0（a1 + 0 を a0 へ）
例：j label → 実際は jal zero, label（戻り先を捨てて無条件ジャンプ）
例：ret → 実際は jalr zero, ra, 0（ra にジャンプし戻り先を捨てる）
AsmWalker での確認：
RISC-V サンプルをコンパイルすると、汎用レジスタパネルに a0(x10) のように ABI 名と番号が並んで表示されます。 call 命令ステップでは a0〜a7 に引数バッジが、ret ステップでは a0 に戻り値バッジが表示されます。

4. Caller-saved と Callee-saved

関数を呼び出す側（Caller）と呼び出される側（Callee）の間で、レジスタ値の「保護義務」を決めたルールです。

種別誰が保存する意味代表例
Caller-saved
（呼び出し側保存）		 関数を呼ぶ前に Caller が待避 Callee は自由に上書きしてよい。呼び出し後も値が必要なら Caller がスタックに退避しておく。 ARM: r0〜r3, r12
x86: rdi, rsi, rax
RV32: a0〜a7, t0〜t6
Callee-saved
（呼び出され側保存）		 関数内部で Callee が退避・復元 Callee は使う前にスタックへ退避し、return 前に必ず復元しなければならない。 ARM: r4〜r11
x86: rbx, rbp, r12〜r15
RV32: s0〜s11, sp

コンパイラはこのルールに従ってコードを生成するため、呼び出し前後でレジスタ値が維持されるか否かが保証されます。アセンブラを読む際は「この命令実行後、このレジスタは変わるか？」を確認するのに役立ちます。

; ARM の例: Callee-saved レジスタ r4 を使う関数
example:
    push {r4, lr}     ; ← Callee が先頭で退避（r4 と戻り先 LR を保存）
    mov  r4, r0       ; r4 に引数を保存して使いまわす
    ...
    pop  {r4, pc}     ; ← return 前に復元（呼び出し元の r4 が壊れない）+ 関数復帰

5. 引数・戻り値レジスタのまとめ

アーキテクチャ第1〜第N引数整数の戻り値最大レジスタ引数数
ARM / AAPCS r0, r1, r2, r3 r0（64bit は r0:r1） 4
x86-64 / SysV rdi, rsi, rdx, rcx, r8, r9 rax（128bit は rax:rdx） 6
RISC-V RV32 a0, a1, a2, a3, a4, a5, a6, a7 a0（64bit は a0:a1） 8

最大引数数を超えた引数はスタック経由で渡されます。-O0 でコンパイルした AsmWalker のサンプルは引数が少ないため、すべてレジスタで渡されます。
