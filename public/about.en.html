About AsmWalker

AsmWalker is a browser-based learning tool for exploring C-to-assembly correspondence, one instruction at a time.

What You Can Do

  • Compile C source code right in the browser and step through the generated assembly
  • View each instruction's meaning, format, and execution effect in the detail panel
  • Visualize register and stack changes in real time
  • See function calls and return values displayed according to the ABI (calling convention)

How Compilation Works

Compilation uses the public API of Compiler Explorer (Godbolt). The C code you type in the editor is sent from your browser to the Godbolt API, and the returned assembly text is parsed and executed inside the tool.

The Godbolt API returns pure assembly text. It does not include address information.

How Addresses Are Displayed

Addresses like 0x401000 shown in the Assembly panel are not assigned by Godbolt — they are virtual addresses generated inside the tool.

Assignment Method

Each address is computed from the architecture's base address plus the instruction index multiplied by a fixed size:

virtual address = base address + instruction index × 4

x86-640x401000instr 0=0x401000, instr 1=0x401004 …
ARM Cortex-M0x08000000instr 0=0x08000000, instr 1=0x08000004 …

Why "Fixed 4 Bytes"?

Actual instruction sizes vary by architecture:

ArchitectureReal instruction sizeHow this tool handles it
x86-641–15 bytes (variable)Fixed 4 bytes
ARM Thumb22 or 4 bytes (mixed)Fixed 4 bytes
Computing the real byte size of each instruction would require encoding and disassembling every instruction. This tool uses a fixed 4-byte virtual stride to prioritize ease of learning. The step-execution engine keeps the return address of a call instruction consistent with the displayed address of the following instruction.

How Step Execution Works

The tool parses the assembly text received from Godbolt with its own parser and generates an array of snapshots — instruction sequences paired with virtual machine state (registers, stack, flags) at each step. Stepping forward or backward simply references the next or previous snapshot in this array; no re-execution takes place.

Supported instructions (x86-64): MOV / LEA / PUSH / POP / ADD / SUB / IMUL / IDIV / AND / OR / XOR / SHL / SHR / CMP / TEST / JMP / Jcc / CALL / RET / LEAVE and more

Supported instructions (ARM): MOV / LDR / STR / ADD / SUB / MUL / CMP / B / BL / BX / PUSH / POP / LSL / LSR and more
