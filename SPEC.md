# AsmWalker — プロジェクト仕様書

> 最終更新: フェーズ1プロトタイプ完成後、Claude.aiでの壁打ちセッションを経て更新

---

## プロジェクト概要

C言語学習者・組み込みエンジニア向けに、アセンブラ命令のステップ実行とスタック・レジスタの動きをブラウザ上で可視化する学習ツール。

**開発動機**: アセンブラ学習時にスタックとレジスタの動きを直感的に理解できるツールが存在しないという課題から出発。特にARM Cortex-M（組み込み業務で最も使われるアーキテクチャ）に特化した可視化ツールは世界的にも存在しない。

---

## ターゲットユーザー

- C言語を学び始めた学生・エンジニア（アセンブラを初めて学ぶ人）
- ARM Cortex-M系（STM32, nRF52 等）を扱う組み込みエンジニア
- x86とARMの違いを比較学習したい人

## プリセット詳細仕様

### x86-64 / ARM 共通プリセット（5種）

| プリセット名 | Cコードの概要 | 主な学習ポイント |
|---|---|---|
| 関数呼び出し | `main → add(3,5)` | call/BL、引数渡し(EDI/ESI vs R0/R1)、戻り値(EAX vs R0)、戻りアドレスの保存方式の違い |
| 四則演算 | a+b, a-b, a*b, a/b | 2オペランド vs 3オペランド、idiv前のRDX初期化（バグの温床）、SDIV |
| 条件分岐 | `if(a<b)` `if(a==0)` | cmp/CMP+フラグ更新、jl/BLT、test+jz vs CBZ（Thumb-2固有） |
| ポインタとアドレス | `ptr=&x; *ptr=100; y=*ptr` | lea vs ADD SP、&varはアドレス取得、*ptrは「アドレス取得→その先を読み書き」の2ステップ |
| 配列を関数に渡す | `sum_array(a, 3)` | 先頭アドレスのみ渡される（コピーなし）、arr[i]=[rdi+i*4]のアドレス計算 |

### ARM Cortex-M 専用プリセット（1種）

| プリセット名 | 概要 | 主な学習ポイント |
|---|---|---|
| 割り込みスタック退避 | `TIM2_IRQHandler` | 8レジスタHW自動退避（アセンブラ命令なし）、EXC_RETURN値のビット構造、Thread/Handler Mode、ISRはC関数として書ける理由 |

## Cortex-M 割り込み仕様（実装済み）

- ハードウェア自動退避レジスタ（例外スタックフレーム）: `R0, R1, R2, R3, R12, LR, PC, xPSR`（8レジスタ = 32バイト）
- 積み順（高アドレスから低アドレスへ）: `xPSR → PC → LR → R12 → R3 → R2 → R1 → R0`
- EXC_RETURN 値:
  - `0xFFFFFFF9`: Thread Mode / MSP使用（ベアメタル最一般的） ← **実装済み**
  - `0xFFFFFFFD`: Thread Mode / PSP使用（RTOS）
  - `0xFFFFFFF1`: Handler Mode / MSP使用（ネスト割り込み）
- HWステップ: `type: 'hw'` フラグで区別し、UI側で橙色ハイライトと「ハードウェアが自動実行 — アセンブラ命令は存在しません」バナーを表示

## 既存ツールとの差別化

| ツール | 差分 |
|---|---|
| AsmWalker (asm.diveintosystems.org) | ARM非対応・C対応表示なし |
| Compiler Explorer (godbolt.org) | ステップ実行なし・スタック可視化なし |
| VisUAL | デスクトップアプリ・C対応なし・Cortex-M特化なし |

**このツールの独自価値**: Cortex-M Thumb-2特化 + ステップ実行 + Cソース対応ハイライト + 日本語説明 + 学習ガイド + ポインタ/配列の可視化 + スタックフレーム色分け図 = 世界的に類似ツールなし
