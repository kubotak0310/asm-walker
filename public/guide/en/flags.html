Flag Registers

Flag registers are special registers that record properties of the last computation result, one bit per property. Conditional branch instructions read these flags to decide whether to jump. ARM and x86 have flag registers; RISC-V does not.

1. ARM Flags (CPSR)

ARM Cortex-M / CPSR

ARM stores four condition flags in the top bits of the CPSR (Current Program Status Register). They are updated by instructions such as ADDS, SUBS, CMP, and TST, and read by conditional branches like BEQ and BLT.

FlagNameSet whenTypical use
N Negative Negative flag Bit 31 of the result is 1 (result is negative in signed arithmetic) BMI (branch if minus)
Z Zero Zero flag Result is exactly 0 BEQ (equal) · BNE (not equal)
C Carry Carry flag Addition produced a carry out of bit 31, or subtraction did not borrow BCS (carry set) · unsigned comparisons (BHI/BLS)
V Overflow Overflow flag Signed result fell outside −2³¹ to 2³¹−1 (positive + positive = negative, or negative + negative = positive) BVS (overflow) · signed comparisons (BGT/BLE)
ARM's C flag is inverted relative to x86:
For ARM subtraction (SUB/CMP), the C flag is 1 when no borrow occurred. x86's CF is 1 when a borrow did occur — the opposite convention. AsmWalker's Special Registers panel shows the live C flag value as you step.

The S suffix for flag updates

In ARM, appending S to an instruction name causes it to update the flags. Without S, flags are unchanged.

MOV  r0, #0     ; r0 = 0. Flags are NOT updated.
MOVS r0, #0     ; r0 = 0. Result is 0, so Z=1, N=0 are written.

CMP  r0, r1     ; Compute r0 - r1 and update flags only (no S needed — CMP always updates)
BEQ  label      ; Jump if Z=1 (i.e. r0 == r1)

Compile the Conditional Branch sample (ARM) and step through it. At the CMP instruction you will see the flags update in the Special Registers panel. The following BEQ/BNE step shows which flag was read to determine the jump.

2. x86-64 Flags (RFLAGS)

x86-64 / RFLAGS

x86-64 stores flags in the 64-bit RFLAGS register. The four or five flags that appear most often in assembly code are listed below. They are updated by CMP, TEST, ADD, SUB, etc., and read by conditional jumps like JE and JL.

FlagNameSet whenTypical use
ZF Zero Flag Result is exactly 0 JE (equal) · JNE (not equal)
SF Sign Flag Most-significant bit of the result is 1 (negative number) JS (negative) · combined with OF for signed comparisons
CF Carry Flag Unsigned overflow — carry out of addition or borrow from subtraction JB (unsigned below) · JAE (unsigned above or equal)
OF Overflow Flag Signed overflow (positive + positive = negative, or negative + negative = positive) JO (overflow) · signed comparisons (JL/JGE)
PF Parity Flag Number of set bits in the low byte of the result is even Floating-point comparisons · some crypto code (rarely used in everyday code)

CMP vs TEST

CMP  rax, rbx    ; compute rax - rbx, update flags only (rax unchanged)
TEST rax, rax    ; compute rax AND rax, update flags only (zero-check idiom)

; Conditional jump examples
cmp  eax, 0      ; eax - 0 → ZF=1 if eax is 0
je   .zero       ; jump to .zero if ZF=1 (i.e. eax == 0)

test eax, eax    ; eax & eax → ZF=1 if eax is 0 (same effect, 1 byte shorter)
jne  .nonzero    ; jump to .nonzero if ZF=0 (i.e. eax != 0)
Try it in AsmWalker:
Compile the Conditional Branch sample (x86) and step through it. At the CMP/TEST instruction, the flags (ZF/SF/CF/OF) update in the Special Registers panel. The next JE/JL step shows whether the branch is actually taken.

3. RISC-V: No Flag Register

RISC-V RV32 / No flags
RISC-V has no flag register.

ARM and x86 use a two-step approach: "compare instruction sets flags → conditional jump reads flags." RISC-V handles this in a single instruction that compares and branches simultaneously.
ArchitectureImplementing "jump if a == b"
ARM CMP r0, r1 (compute r0 − r1 to set flags)
BEQ label (jump if ZF=1)
x86-64 CMP eax, ebx (compute eax − ebx to set flags)
JE label (jump if ZF=1)
RISC-V beq a0, a1, label (compare a0 == a1 and branch — one instruction)

All RISC-V branch instructions (beq, bne, blt, bge, bltu, bgeu) follow the pattern "compare two registers and jump." Because no flags are involved, the Special Registers panel in AsmWalker shows no flags section when RISC-V is selected.

InstructionConditionMeaning
beq rs1, rs2, labelrs1 == rs2Branch if Equal
bne rs1, rs2, labelrs1 != rs2Branch if Not Equal
blt rs1, rs2, labelrs1 < rs2 (signed)Branch if Less Than
bge rs1, rs2, labelrs1 ≥ rs2 (signed)Branch if Greater or Equal
bltu rs1, rs2, labelrs1 < rs2 (unsigned)Branch if Less Than Unsigned
bgeu rs1, rs2, labelrs1 ≥ rs2 (unsigned)Branch if Greater or Equal Unsigned
Comparing against zero:
RISC-V's zero register (always 0) makes zero comparisons natural.
beq a0, zero, label → jump if a0 == 0 (equivalent to ARM's CBZ or x86's test eax,eax + je)
blt a0, zero, label → jump if a0 is negative (equivalent to ARM's BMI)

4. Flags and Conditional Branches

In ARM and x86, a conditional branch reads the flags left by the most recent comparison or arithmetic instruction and decides whether to jump.

ARMx86-64Condition (C notation)Flags checked
BEQJEa == bZ=1
BNEJNEa != bZ=0
BLTJLa < b (signed)N != V
BLEJLEa ≤ b (signed)Z=1 or N != V
BGTJGa > b (signed)Z=0 and N == V
BGEJGEa ≥ b (signed)N == V
BLO / BCCJBa < b (unsigned)C=0 (ARM) / CF=1 (x86)
BHS / BCSJAEa ≥ b (unsigned)C=1 (ARM) / CF=0 (x86)

Signed vs unsigned comparisons:
"Less than" has two variants — signed (BLT/JL) and unsigned (BLO/JB) — which check different flags. The compiler uses BLT/JL for C's int (signed) and BLO/JB for unsigned int. When reading assembly, always check which variant is used.
