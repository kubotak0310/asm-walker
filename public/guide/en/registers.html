Register Reference

Registers are the fastest storage locations a CPU can directly operate on. Every computation, memory access, and jump goes through registers. This page lists the registers for ARM, x86-64, and RISC-V RV32, together with their ABI roles.

Thumb mode and the frame pointer:
Cortex-M uses the Thumb instruction set. Because Thumb-2 two-register instructions (like push/pop) have limited access to r8 and above, GCC uses r7 as the frame pointer with -mthumb, and r11 in ARM mode. AsmWalker samples compile in Thumb mode, so you will see r7 used as FP.
Try it in AsmWalker:
Compile the Function Call sample and step to the bl instruction. You will see argument badges on r0–r3 showing the values being passed. At the bx lr / pop {pc} step, a "return" badge appears on r0.

Writing the 32-bit name zeros the upper 32 bits automatically:
When passing an int (32-bit) argument, GCC uses edi rather than rdi. The x86-64 specification guarantees that writing to edi clears the upper 32 bits of rdi, so the effect is identical to writing the full 64-bit register.
Try it in AsmWalker:
Compile an x86 sample and step to the call instruction. Argument badges will appear on rdi, rsi, rdx, etc. At the ret step, a "return" badge appears on rax (or eax).

How the zero register (x0) is used:
Having a dedicated always-zero register lets many pseudo-instructions be expressed simply.
e.g. mv a0, a1 → actually addi a0, a1, 0 (a1 + 0 into a0)
e.g. j label → actually jal zero, label (jump, discarding the return address)
e.g. ret → actually jalr zero, ra, 0 (jump to ra, discarding the return address)
Try it in AsmWalker:
Compile a RISC-V sample and watch the Register panel — registers are displayed as a0(x10), showing both the ABI name and hardware number side by side. At the call step, argument badges appear on a0–a7; at the ret step, a "return" badge appears on a0.

4. Caller-saved vs Callee-saved

The ABI defines which side — the caller (the function making the call) or the callee (the function being called) — is responsible for preserving a register's value across a call.

CategoryWho saves itMeaningExamples
Caller-saved Caller saves before the call The callee may freely overwrite these. If the caller needs the value after the call, it must spill it to the stack first. ARM: r0–r3, r12
x86: rdi, rsi, rax
RV32: a0–a7, t0–t6
Callee-saved Callee saves inside the function The callee must spill these to the stack at the start and restore them before returning. ARM: r4–r11
x86: rbx, rbp, r12–r15
RV32: s0–s11, sp

The compiler generates code that follows these rules, so you can reason about whether a register is safe to read after a function call. When reading assembly, ask "could this call have changed this register?"

; ARM example: function that uses callee-saved register r4
example:
    push {r4, lr}     ; callee saves r4 and LR at the top
    mov  r4, r0       ; keep the argument in r4 across inner calls
    ...
    pop  {r4, pc}     ; restore r4 and return (caller's r4 is untouched)
