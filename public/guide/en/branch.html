1. What is the Flag Register?

In addition to the general-purpose registers (rax, r0, etc.), CPUs have a special register called the flag register (APSR on ARM / RFLAGS on x86). After every arithmetic or comparison instruction, bits in this register are automatically set to describe the nature of the result.

Conditional branch instructions read these flags to decide whether to branch. This means a C condition like if (a > b) is implemented in assembly as a two-step process: "compare instruction sets the flags → branch instruction reads the flags and jumps (or not)."

3. Instructions That Set Flags: CMP and TEST

CMP — Comparison

CMP a, b computes a - b and sets the flags based on the result. The result itself is discarded — it is a subtraction used only to set flags.

ARM — CMP flags

; CMP 3, 3  (a == b)
; → Z=1, N=0, C=1, V=0

; CMP 5, 3  (a > b)
; → Z=0, N=0, C=1, V=0

; CMP 3, 5  (a < b)
; → Z=0, N=1, C=0, V=0

ARM C flag: 1 means "no borrow".
a >= b → C=1, a < b → C=0

x86-64 — CMP flags

; CMP 3, 3  (a == b)
; → Z=1, N=0, C=0, V=0

; CMP 5, 3  (a > b)
; → Z=0, N=0, C=0, V=0

; CMP 3, 5  (a < b)
; → Z=0, N=1, C=1, V=0

x86 CF: 1 means "borrow occurred".
a < b → CF=1, a >= b → CF=0

ARM and x86 C flags have opposite meanings
For the same CMP 3, 5 (a < b), ARM gives C=0 while x86 gives CF=1. Switch between ARM and x86 modes in AsmWalker and compare how the flags change after a CMP instruction.

TEST — Bit Mask Check

TEST a, b computes a AND b and sets the flags (discarding the result). It is mainly used to check whether a particular bit is set. 

; Check if (x != 0)
test eax, eax     ; eax AND eax → Z=1 if eax is 0
jnz  .nonzero     ; branch if Z=0 (eax ≠ 0)

; Check if (x & 1) — odd number test
test eax, 1       ; eax AND 1 → Z=1 if bit 0 is clear
jz   .even        ; branch if Z=1 (even)
Try it in AsmWalker
After stepping through a CMP instruction, the Instruction Detail panel shows the flag changes as flags: Z=0, N=0, C=0, V=0. Confirm that the following conditional branch instruction reads those flags to decide whether to branch.

4. Conditional Branch Instructions (Jcc / Bcc)

After CMP sets the flags, a conditional branch instruction reads them and decides whether to jump to the specified label. If the condition is not met, execution falls through to the next instruction.

How to read the instruction names
x86: J = Jump, E=Equal, N=Not, L=Less, G=Greater, B=Below (unsigned), A=Above (unsigned).
ARM: B = Branch, with suffixes like EQ=Equal, NE=Not Equal, etc.

5. How if / else Is Translated

Let's see how GCC translates an if/else statement.

// C
if (a > b) {
    result = a;
} else {
    result = b;
}

ARM — if/else translation

    ldr  r0, [fp, #-8]    ; r0 = a
    ldr  r1, [fp, #-12]   ; r1 = b
    cmp  r0, r1            ; set flags from a - b
    ble  .Lelse            ; if a <= b, jump to else

; if body (a > b)
    ldr  r3, [fp, #-8]    ; result = a
    str  r3, [fp, #-16]
    b    .Lend             ; skip else

.Lelse:
    ldr  r3, [fp, #-12]   ; result = b
    str  r3, [fp, #-16]

.Lend:

x86-64 — if/else translation

    mov  eax, DWORD PTR [rbp-4]  ; eax = a
    cmp  eax, DWORD PTR [rbp-8]  ; set flags from a - b
    jle  .Lelse                   ; if a <= b, jump to else

; if body (a > b)
    mov  eax, DWORD PTR [rbp-4]
    mov  DWORD PTR [rbp-12], eax  ; result = a
    jmp  .Lend                    ; skip else

.Lelse:
    mov  eax, DWORD PTR [rbp-8]
    mov  DWORD PTR [rbp-12], eax  ; result = b

.Lend:

GCC's basic pattern is to generate a branch with the negated condition. For if (a > b), it generates a jump "when a > b is false (i.e., a <= b)" to the else block. When the condition is true, execution simply falls through to the next instruction.

Getting comfortable with "inverted" conditions
C's if (cond) → assembly generates j[NOT cond] else_label. Since it's "the condition to jump to else," it can feel backwards at first. Think of it as: "this branch is an escape hatch to the else block."
Try it in AsmWalker
At a conditional branch step, observe whether the assembly panel highlight moves to the next line (fall-through) or jumps to the label. The Instruction Detail panel shows an explanation like "branched because Z=1."

6. How while / for Loops Are Translated

A loop translates to: "check condition → body → unconditional jump back to top."

// C
int i = 0;
int sum = 0;
while (i < 5) {
    sum += i;
    i++;
}

ARM — while loop

    mov  r3, #0
    str  r3, [fp, #-8]    ; i = 0
    mov  r3, #0
    str  r3, [fp, #-12]   ; sum = 0

.Lloop:
    ldr  r3, [fp, #-8]    ; r3 = i
    cmp  r3, #4            ; flags from i - 4
    bgt  .Lend             ; if i > 4, exit

    ldr  r2, [fp, #-12]
    ldr  r3, [fp, #-8]
    add  r3, r2, r3        ; sum += i
    str  r3, [fp, #-12]

    ldr  r3, [fp, #-8]
    add  r3, r3, #1        ; i++
    str  r3, [fp, #-8]

    b    .Lloop             ; back to top
.Lend:

x86-64 — while loop

    mov  DWORD PTR [rbp-4], 0   ; i = 0
    mov  DWORD PTR [rbp-8], 0   ; sum = 0

.Lloop:
    cmp  DWORD PTR [rbp-4], 4   ; flags from i - 4
    jg   .Lend                  ; if i > 4, exit

    mov  eax, DWORD PTR [rbp-4]
    add  DWORD PTR [rbp-8], eax ; sum += i

    add  DWORD PTR [rbp-4], 1   ; i++

    jmp  .Lloop                 ; back to top
.Lend:

With -O0, GCC typically uses "check at the top, then execute the body." With -O1 or higher, it may convert to a do-while form — "execute body first, then check" — which eliminates one redundant jump.

Try it in AsmWalker
Step through and confirm that the same loop body instructions are highlighted multiple times. At the conditional branch, observe both "did not branch this iteration" and "branched (exited loop)" and correlate with the flag values.

7. Signed vs. Unsigned Comparison

Whether a C comparison is signed or unsigned depends on the variable's type. Assembly uses different branch instructions for each.

TypeKind of comparisonx86 branchARM branch
int (signed)Signed comparisonJL / JG / JLE / JGEBLT / BGT / BLE / BGE
unsigned intUnsigned comparisonJB / JA / JBE / JAEBCC / BHI / BLS / BCS

This reflects how the CPU uses flags differently. Signed comparisons rely on the relationship between the N and V flags, while unsigned comparisons use the C and Z flags.

Why the distinction matters
-1 in two's-complement is 0xFFFFFFFF. Interpreted as unsigned, that's about 4 billion (4,294,967,295). -1 < 1 is true in signed arithmetic, but false in unsigned. The compiler automatically selects the correct comparison instruction based on the C variable type.

8. Logical Operators and Short-Circuit Evaluation

Compound conditions using && or || expand into multiple CMP + branch sequences in assembly. 

// C: if (a > 0 && b > 0)
    cmp  eax, 0
    jle  .Lfalse    ; if a <= 0, bail immediately (short-circuit: skip b)
    cmp  ebx, 0
    jle  .Lfalse    ; if b <= 0, bail

; both are true
    ...
.Lfalse:
    ...

C's logical AND (&&) evaluates left-to-right and stops as soon as it finds a false value — this is called short-circuit evaluation. In assembly, this is achieved by jumping to a label immediately when the left-hand side is false.

ARM — OR condition (||)

    cmp  r0, #0
    bgt  .Ltrue    ; if a > 0, true immediately
    cmp  r1, #0
    ble  .Lfalse   ; if b <= 0, false
.Ltrue:
    ; at least one is true

x86-64 — OR condition (||)

    cmp  eax, 0
    jg   .Ltrue    ; if a > 0, true immediately
    cmp  ebx, 0
    jle  .Lfalse   ; if b <= 0, false
.Ltrue:
    ; at least one is true
