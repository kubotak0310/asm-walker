How to Read Assembly

Assembly can look intimidating at first, but once you grasp the basic rules it becomes readable. Let's go through the most common instructions one by one, comparing them to their C equivalents.

1. Basic Instruction Structure

Each line of assembly consists of a mnemonic (instruction name) and operands (targets).

MOV Mnemonic
(what to do)

(destination)

5 2nd operand

Mnemonics are abbreviations of English words. MOV = Move, ADD = Add, STR = Store Register, etc.

Tip for remembering mnemonics:
Think of the full English word: MOV=Move, CMP=Compare, LDR=Load Register, STR=Store Register, BL=Branch with Link (subroutine call).

Instruction length and address display

Each instruction occupies multiple bytes in memory. The byte count varies by instruction and architecture.

2. Reading Direction: Left is Destination, Right is Source

Both architectures supported by this tool use destination ← source (left ← right) order.

ArchitectureExampleMeaning
x86-64 (Intel syntax) mov rax, 5 Store 5 into rax
ARM mov r0, #5 Store 5 into r0
Difference from AT&T syntax (GCC default on Linux):
Running gcc -S on Linux produces AT&T syntax where the order is reversed (right → left). movl $5, %eax means the same thing but reads in the opposite direction. AsmWalker uses Intel syntax (-masm=intel), so left is always the destination.

3. What Are Registers?

Registers are ultra-fast variables inside the CPU itself. While C variables live in memory, registers sit on the CPU chip and are orders of magnitude faster to access.

eax and rax are the same register:
rax (64-bit) contains eax as its lower 32 bits, ax as its lower 16 bits, and al as its lower 8 bits. When GCC handles an int (32-bit) value, it uses eax. Writing to eax automatically zeroes the upper 32 bits of rax.

4. MOV — Moving Values

The most common instruction. Used for register-to-register copies, loading immediate values, and reading from / writing to memory.

ARM

mov r0, #3       ; r0 = 3 (immediate)
mov r1, r0       ; r1 = r0 (register copy)

ARM prefixes immediate values with #.

x86-64

mov eax, 3       ; eax = 3 (immediate)
mov ebx, eax     ; ebx = eax (register copy)

x86 Intel syntax does not use # for immediates.

Mapping to C:

int a = 3;   →   mov eax, 3        ; eax corresponds to variable a
int b = a;   →   mov ebx, eax      ; ebx corresponds to variable b
Try it in AsmWalker:
At a MOV step, the Instruction Detail panel shows the value flow as rax ← 3. The changed register also lights up white in the Registers panel.

5. ADD / SUB — Addition and Subtraction

Adds or subtracts two values and writes the result back to the first operand.

ARM

add r0, r1, r2   ; r0 = r1 + r2
sub r0, r0, #1   ; r0 = r0 - 1

ARM often uses 3-operand form (explicit destination).

x86-64

add eax, edx     ; eax = eax + edx
sub eax, 1       ; eax = eax - 1

x86 uses 2-operand form (destination = destination OP source).

; C: int result = a + b;
; x86-64 expansion:
mov  edx, DWORD PTR [rbp-4]   ; edx = a (load from memory)
mov  eax, DWORD PTR [rbp-8]   ; eax = b
add  eax, edx                  ; eax = a + b
Try it in AsmWalker:
At an ADD step, the Instruction Detail panel shows eax ← eax(3) + edx(5) = 8 — values before and after the operation.

6. Memory Access — LDR/STR (ARM) and [ ] (x86)

Registers are temporary storage inside the CPU, so variables need to be read from / written to memory for persistence.

Load (read from memory)

ARM — LDR (Load Register)

ldr r0, [r7, #4]  ; r0 = memory[r7+4]

[ ] contains the memory address (base + offset).

x86 — MOV with [ ]

mov eax, DWORD PTR [rbp-4]
; eax = memory[rbp-4] (read 4 bytes)

DWORD PTR specifies a 4-byte access width.

Store (write to memory)

ARM — STR (Store Register)

str r0, [r7, #4]  ; memory[r7+4] = r0

The reverse of LDR — saves r0's value to memory.

x86 — MOV with [ ] (memory on the left)

mov DWORD PTR [rbp-4], eax
; memory[rbp-4] = eax (write 4 bytes)

x86 uses MOV for both loads and stores; direction is set by which side has [ ].

How to read [ ] expressions:
[rbp-4] or [r7, #8] are address calculation expressions. rbp-4 means "the address obtained by subtracting 4 from rbp's value". Local variables are placed at fixed offsets from the frame pointer, so each new variable gets a new offset (-4, -8, -12…).

Memory size specifiers (x86)

SpecifierSizeC type
BYTE PTR1 bytechar

Try it in AsmWalker:
At an LDR/STR (ARM) or mov [...], reg (x86) step, the corresponding cell in the Stack panel highlights and you can see the value change.

7. CMP — Compare

CMP subtracts two values but does not store the result — it only updates the flags.

cmp eax, 5    ; compute eax - 5 and update flags (eax unchanged)
je  .L1       ; if Zero Flag is set, jump to .L1 (eax == 5)

The flag register records properties of the last computation result as individual bits.

FlagMeaningSet when
Zero Flag (ZF)Result is zerocmp eax, eax (equal values)
Negative/Sign Flag (NF/SF)Result is negativecmp eax, 10 where eax < 10
Carry Flag (CF)Unsigned overflowUnsigned arithmetic overflow
Overflow Flag (OF/VF)Signed overflowSigned arithmetic overflow

; C: if (a > 0) { ... }
cmp eax, 0    ; compute flags for a - 0
jle .L_else   ; if a <= 0, jump to else
; ↑ body of if
.L_else:
Reading tip:
Compilers typically generate a branch on the inverted condition of the C test — if (a > 0) becomes "jump if NOT (a > 0)". Think of the jump as an escape to the else block, and the body of the if falls through naturally.
Try it in AsmWalker:
At a CMP step, the Instruction Detail panel shows which flags were updated (ZF / NF, etc.). At the next conditional branch step, you can see whether the jump actually fires.

8. PUSH / POP — Stack Operations

PUSH pushes a value onto the stack; POP retrieves it. The stack pointer (SP / RSP) moves automatically.

ARM

push {r7, lr}  ; save r7 and lr to the stack
               ; sp -= 8

pop  {r7, pc}  ; restore r7 and pc
               ; sp += 8

Multiple registers can be saved/restored in one instruction.

x86-64

push rbp       ; [rsp-8] = rbp, rsp -= 8

pop  rbp       ; rbp = [rsp], rsp += 8

Operates on one register at a time.

PUSH/POP always appear in function prologues and epilogues: "save the caller's register values so we don't clobber them, restore them before returning."

Try it in AsmWalker:
At a PUSH step, a new cell appears at the bottom of the Stack panel and SP moves down. At a POP step, the cell disappears and SP moves back up.

9. Tips for Reading Assembly

  1. Read left → right as "put what into where" (Intel syntax / ARM both use this order)
  2. [ ] means memory access. The expression inside is an address calculation.
  3. Mnemonics are abbreviated English words. When you see an unfamiliar instruction, expand the abbreviation (MOV=Move, LDR=Load Register).
  4. CMP is always followed by a conditional jump. The condition is often the inverse of what you expect.
  5. Function entry and exit follow a fixed pattern (prologue: push rbp / mov rbp, rsp; epilogue: pop rbp / ret).
  6. Only ARM uses # before immediates. Intel syntax for x86 does not.
  7. You don't have to read every line. Just trace the register that holds the variable you care about.
